## 变化

### [2026-02-11]

最近，快春节了，重构了 hzDocs 这个文档网站，主要的变化是：

- 升级了新版本 fumadocs。由于大版本跳动，fumadocs 和 nextjs 的变化都太大，所以我是推倒后重新建立的。
  - 这就是为什么我几乎不做前端的代码，每一年你的代码都会完全失效，无法渐进更新。
  - 这一次 fumadocs 累积改进了很多东西，对自建文档站的人更友好了，相比去年我首次搭建省力多了。
  - 上次的网站开发时候改写的代码很多，几乎是翻尽了 fuma 当时的源代码去寻找 hack 的各种方法，来满足我的需求，但今次很多都省掉了。
  - 最大的变化，一是基础代码支持跳转到 LLM 问答，另一个是主题更多，所以我换了一个。
- 由于完全重写，所以以往有的细节可能就没能填上了
  - 上一版中有一个 blog 子分区，是实验性的；今次摘除了，既然 [hedzr.com](https://hedzr.com/) 一直工作良好
  - 继续支持方向键翻页功能，暂时没有发现问题 
- 现在文档页面乃至于每一段文字都支持提供 feedback 到 github Discussions 中
  - 当前是匿名的。暂无计划做双选（匿名或者 github 账户登录）
  - 对于有心人，可以匿名发一句，然后跟进到讨论区中以 github 身份继续追加发言
- 因为今次省掉了很多 hacks，所以理论上网页的渲染速度有相当的提升，因为无需额外多一次或者数次 PageTree metadatas 的扫描和分组排序了。

#### cmdr

> 当前为 `cmdr.v2` 版本

cmdr 是一个命令行参数处理框架，同时还提供应用程序设置集的集成能力。

所以当你在进行终端应用程序的开发时，cmdr 能够提供初始环境的处理功能。
当你使用 cmdr 的方式建立了子命令、命令行选项的定义之后，你的精力可以专注于实现具体功能。

cmdr 也是一个跨语言的软件库。我们用几种开发语言（Golang，C++，C# 等等）实现了 cmdr 统一的界面。
这些软件库在不同语言环境中提供基本一致的编程界面以及相似的终端用户体验。

#### Store

[Store](https://github.com/hedzr/store) 被用于集成到 cmdr 中提供配置管理功能。

同时 Store 也提供一个通用的、高性能的层次性数据的键值对管理，所以你可以将其视为内存高速缓存，Redis-like，或者 httprouter 路由解析工具，等等。

Store 支持键值对的自动清除（TTL）功能。

Store 也支持内建的 `/[id]/` 路由解析和匹配功能。

#### is

[is](https://github.com/hedzr/is) 是一个环境检测函数库。

它提供大量运行环境（以及构建环境）的检测函数，你可以用 `is.DebugMode()` 这样的语法来进行测试。

#### progressbar

[progressbar](https://github.com/hedzr/progressbar) 是一个完全无依赖（好像是这样）的控制台进度条。

初始的意图是为了复刻 python install 时候的进度条外观，后来则集成了大量的其他风格，划分为 Stepper 和 Spinner 两类。

然后在此基础上，progressbar 提供了一些包装方法来方便你将后台任何与进度条相互结合，尤其是针对分步骤的多线绪任务。

### VSCode Extensions

#### Go Main Runner

基本上，[Go Main Runner](https://github.com/hedzr/vscode-ext-go-main-runner) 只做一件事：就地运行 golang 项目中的 main 函数而无需移开鼠标、或者进行命令行输入。

#### Zig Main runner

类似地，[Zig Main Runner](https://github.com/hedzr/vscode-ext-zig-main-runner) 是为了就地运行 zig 项目中 main 函数。

#### Remarks

这两个 VSCode 插件都包含一些细微的调整，目的在于简化生活。

例如 Go Main Runner 包含到官方 Golang 插件的依赖，所以打开一个 vscode workspace 之后 Enable Go Main Runner 你就能自动为工作区使能 Golang 相关的插件支持。你并不需要额外的其他花哨的插件，对于 Golang 开发来说，其它的插件往往意义不大——但 Go Main Runner 或许能够成为例外。

当然，取决于开发内容，这里所说的其他插件不包含语法高亮、formatters 之类的插件，我主要是指类似于 Go Mod List 之类似乎有用、似乎无用的插件。至于 Go Main Runner 本身是不是有用，也是取决于你的项目是有很多 main 函数呢，还是只有一个 cli 入口。

### NextJS/React Libraries

#### cmdr-docs-dev

[cmdr-docs-dev](https://github.com/hedzr/cmdr-docs-dev) 是本 docsite 的源代码 repo。

它是使用 NextJS + Fumadocs + Tailwind 构造的，主要是利用了 MDX 的核心功能来建立基于 Markdown 的 docsite。

技术性不强。

但是考虑到事前数年来的对 docsite generators/makers 的挑选、评估，加上在这个框架上投入了相当的精力来做整合，所以我个人认为可能它对你或许是具有启发性的。

### Shell Frameworks

#### bash.sh

[bash.sh](https://github.com/hedzr/bash.sh) 包含了最必要的基础函数集，并将它们组织为一种有利于自由扩展的单一主文件结构，可以被视为是做 ops shell 脚本开发的较好的起点。

单纯在 bash.sh 中查证基础环境探测技术也是有益的。
